# P1 需求文档

## 1. 负责模块

- `src/main.c`
- `src/storage.c` / `include/storage.h`
- `src/utils.c` / `include/utils.h`
- `include/model.h`

## 2. 目标简介

P1 负责项目整体架构设计与基础模块搭建，确立系统分层结构、数据存储方案、公共工具接口，并为后续 P2-P5 业务模块提供基础支持。

## 3. 主要功能

### 3.1 系统架构设计

- 明确项目目录结构：`src/`、`include/`、`data/`、`docs/`、`build/`
- 定义模块分层：数据层、业务层、展示层、工具层
- 确立模块之间的依赖关系和接口边界

### 3.2 数据模型定义

- 在 `include/model.h` 中定义核心数据结构
- 包括：物品信息、借用记录、归还记录
- 明确数据字段与关联关系

### 3.3 数据持久化基础

- 在 `src/storage.c` / `include/storage.h` 中设计 TXT 文件读写接口
- 定义数据文件路径和格式
- 支持加载、保存、容错处理

### 3.4 公共工具实现

- 在 `src/utils.c` / `include/utils.h` 中提供共用工具函数
- 包括：输入读取、字符串处理、日期验证、错误提示
- 提高模块复用性、降低重复代码

### 3.5 程序入口与流程控制

- 在 `src/main.c` 中实现程序入口
- 执行模块初始化、数据加载、主菜单循环、退出保存
- 提供整体流程控制接口给后续模块调用

## 4. 与其他模块协作

- 为 P2、P3、P4、P5 提供统一的数据模型和基础接口
- P2 依赖 `model`、`storage`、`utils` 实现业务逻辑
- P3 依赖 `storage`、`utils` 与 `main` 进行交互集成
- P4 依赖 `model` 与 `main` 完成公共底层支持
- P5 依赖所有模块完成整体集成与调度

## 5. 交付内容

- `docs/p1.md`
- `src/main.c`
- `include/model.h`
- `src/storage.c`、`include/storage.h`
- `src/utils.c`、`include/utils.h`
- 模块接口说明与初始化流程文档

## 6. 重点说明

- P1 主要承担基础架构与模块接口，不负责具体业务功能实现。
- 任何业务功能都应依赖已定义的基础模块与数据模型。
- P1 输出的接口必须清晰、稳定，以便后续 P2-P5 快速对接。
