# 实验室物品出入账管理系统 项目规范文档

## 1. 项目概述

本项目旨在构建一套实验室物品出入账管理系统，支持实验室物品信息管理、借用登记、归还登记、查询统计与数据持久化。系统采用命令行菜单式交互，使用 TXT 文本文件进行数据存储，强调模块化设计与可维护性。

## 2. 系统目标与设计原则

- 目标：实现实验室物品的统一管理，提升借用归还流程规范性与查询统计效率。
- 设计原则：模块分层、数据持久化、输入校验、用户友好、易扩展。

## 3. 项目结构

- `src/`：C 语言源代码实现。
- `include/`：头文件定义。
- `data/`：TXT 数据文件存放目录。
- `docs/`：项目文档与需求说明。
- `build/`：编译生成的可执行文件与输出。

## 4. 模块划分

### 4.1 数据层

- 主要文件：`include/model.h`、`src/storage.c`、`include/storage.h`
- 职责：定义数据模型、实现数据持久化接口、处理文件加载和保存。
- 核心数据对象：
  - `Item`：物品信息
  - `BorrowRecord`：借用记录
  - `ReturnRecord`：归还记录

### 4.2 工具层

- 主要文件：`src/utils.c`、`include/utils.h`
- 职责：提供公共工具函数，包括安全输入读取、字符串处理、日期验证、错误提示等。

### 4.3 业务层

- 主要文件：`src/item.c`、`include/item.h`、`src/borrow.c`、`include/borrow.h`
- 职责：实现物品管理、借用登记、归还登记、库存校验与记录状态更新。

### 4.4 展示层

- 主要文件：`src/report.c`、`include/report.h`、`src/ui.c`、`include/ui.h`、`src/main.c`
- 职责：实现菜单界面、用户交互、查询显示、报表生成，以及程序入口与整体流程控制。

## 5. 功能需求

### 5.1 物品管理

- 新增物品：录入物品编号、名称、型号、库存数量和描述。
- 修改物品：支持更新物品信息和库存数量。
- 删除物品：删除不再使用的物品信息。
- 查询物品：按编号或名称检索物品并显示详细信息。

### 5.2 借用管理

- 借用登记：记录借用人、物品编号、借用数量、借用日期与预计归还日期。
- 库存检查：借用前检查库存数量，库存不足时拒绝借用。
- 库存更新：借出时减少库存数量。

### 5.3 归还管理

- 归还登记：记录实际归还数量与归还日期。
- 状态更新：更新借用记录状态为已归还或部分归还。
- 库存恢复：归还后按实际归还数量增加库存。

### 5.4 查询与报表

- 支持按物品编号、物品名称、借用人等条件查询。
- 能显示当前库存、借用状态和归还情况。
- 生成功能报表，汇总借用与归还数据，支持出入账统计。

### 5.5 数据持久化

- 采用 TXT 文本文件保存数据。
- 启动时自动加载已有数据，退出或操作变更时保存数据。

### 5.6 用户交互

- 菜单式命令行界面，提供清晰操作提示。
- 对非法输入进行错误提示，增强易用性。

## 6. 非功能需求

- 系统稳定可靠，业务流程正确。
- 界面简洁、操作方便。
- 输入验证严密，避免非法数据导入。
- 数据持久化机制可靠，避免数据丢失。
- 模块化设计，便于后续维护与扩展。

## 7. 数据模型说明

### 7.1 Item

- `id`：唯一整数 ID
- `code`：物品编号
- `name`：物品名称
- `model`：物品型号
- `quantity`：库存数量
- `description`：物品描述

### 7.2 BorrowRecord

- `id`：借用记录 ID
- `user`：借用人姓名
- `item_code`：借用物品编号
- `quantity`：借用数量
- `borrow_date`：借用日期（YYYY-MM-DD）
- `due_date`：预计归还日期（YYYY-MM-DD）
- `status`：借用状态（`BORROW_ACTIVE` 或 `BORROW_RETURNED`）

### 7.3 ReturnRecord

- `id`：归还记录 ID
- `borrow_id`：关联借用记录 ID
- `return_date`：归还日期（YYYY-MM-DD）
- `quantity`：归还数量

## 8. 数据存储规范

### 8.1 文件路径

- `data/items.txt`
- `data/borrow_records.txt`
- `data/return_records.txt`

### 8.2 文件格式

- `items.txt`：`ID,code,name,model,quantity,description`
- `borrow_records.txt`：`ID,user,item_code,quantity,borrow_date,due_date,status`
- `return_records.txt`：`ID,borrow_id,return_date,quantity`

### 8.3 约束

- 字段间用逗号分隔。
- 字段内容中不应包含逗号。
- `status` 字段取值为 `0`（活动借用）或 `1`（已归还）。
- 日期格式统一为 `YYYY-MM-DD`。
- 数量字段为非负整数。

## 9. 关键流程

1. 启动时检查并创建 `data/` 目录与数据文件。
2. 从 TXT 文件加载物品、借用记录与归还记录。
3. 显示主菜单并接收用户操作。
4. 执行业务操作并实时更新内存数据。
5. 变更后保存数据到对应 TXT 文件。
6. 退出时完成所有数据持久化。
