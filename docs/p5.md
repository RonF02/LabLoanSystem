# P5 需求文档

## 1. 负责模块

- `src/main.c`
- 与所有模块协作的整体流程控制

## 2. 目标简介

P5 负责项目总体集成与流程控制，实现程序入口、模块初始化、菜单调度、数据加载与退出保存，确保系统作为一个整体正确运行。

## 3. 主要功能

### 3.1 程序入口

- 编写 `main` 函数，作为程序启动入口。
- 初始化系统环境、加载配置与数据文件。

### 3.2 模块调度

- 初始化子模块：`storage`、`utils`、`item`、`borrow`、`report`、`ui`。
- 实现菜单主循环，接受并分发用户选择。
- 根据用户输入调用对应功能模块。

### 3.3 数据生命周期管理

- 启动时加载所有数据文件到内存模型。
- 运行时调用各模块进行修改、查询、报表生成。
- 退出时保存所有数据，并释放资源。

### 3.4 异常与退出处理

- 处理异常情况，例如配置文件缺失、数据加载失败、模块初始化错误等。
- 提供安全退出流程，确保数据完整性。

## 4. 与其他模块协作

- `main.c` 作为协调层，负责调用所有模块接口。
- 集成 `ui` 的输入输出流程。
- 使用 `storage` 的数据持久化接口。
- 通过 `item` / `borrow` / `report` 提供业务能力。

## 5. 交付内容

- `src/main.c`
- 程序启动与退出流程说明
- 集成测试场景与运行说明
